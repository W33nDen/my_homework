## Опис завдання

Створити Bash-скрипт для автоматичного створення резервних копій (бекапів) файлів логів. Скрипт має перевіряти вхідні аргументи, захищатися від паралельного запуску та створювати архів логів у форматі `.tar.gz` з міткою часу.

---

## Код скрипта `backup.sh`

```bash
#!/bin/bash

# Скрипт бекапу логів
# Використання: ./backup.sh /path/to/logs /path/to/backup

# Перевірка аргументів
if [ $# -ne 2 ]; then
    echo "Помилка: потрібно 2 аргументи"
    echo "Використання: $0 /path/to/logs /path/to/backup"
    exit 1
fi

LOGS_DIR="$1"
BACKUP_DIR="$2"

# Перевірка існування каталогів
if [ ! -d "$LOGS_DIR" ]; then
    echo "Помилка: каталог логів '$LOGS_DIR' не існує"
    exit 1
fi

if [ ! -d "$BACKUP_DIR" ]; then
    echo "Помилка: каталог бекапів '$BACKUP_DIR' не існує"
    exit 1
fi

# Захист від паралельного запуску
LOCK_FILE="/tmp/backup.lock"

if [ -e "$LOCK_FILE" ]; then
    echo "Помилка: скрипт вже виконується (знайдено $LOCK_FILE)"
    exit 1
fi

# Створити lock-файл
touch "$LOCK_FILE"

# Видалити lock-файл при завершенні (навіть при помилці)
trap "rm -f $LOCK_FILE" EXIT

# Формат імені архіву: logs_backup_YYYY-MM-DD_HH-MM.tar.gz
TIMESTAMP=$(date +"%Y-%m-%d_%H-%M")
ARCHIVE_NAME="logs_backup_${TIMESTAMP}.tar.gz"
ARCHIVE_PATH="${BACKUP_DIR}/${ARCHIVE_NAME}"

# Створення архіву
echo "Створення бекапу логів..."
tar -czf "$ARCHIVE_PATH" -C "$LOGS_DIR" .

# Перевірка результату
if [ $? -eq 0 ]; then
    echo "✅ Успіх! Бекап створено: $ARCHIVE_PATH"
    echo "Розмір архіву: $(du -h "$ARCHIVE_PATH" | cut -f1)"
else
    echo "❌ Помилка створення архіву"
    exit 1
fi
```

## Пояснення роботи скрипта

### 1. Перевірка аргументів

```bash
if [ $# -ne 2 ]; then
    echo "Помилка: потрібно 2 аргументи"
    exit 1
fi
```

Скрипт перевіряє, чи передано рівно 2 аргументи (шлях до логів та шлях до бекапів). Якщо ні — виводить помилку та завершується з кодом `1`.

### 2. Перевірка існування каталогів

```bash
if [ ! -d "$LOGS_DIR" ]; then
    echo "Помилка: каталог логів не існує"
    exit 1
fi
```

Перевіряється, чи існують обидва каталоги. Використовується тест `-d` (directory).

### 3. Захист від паралельного запуску

```bash
LOCK_FILE="/tmp/backup.lock"

if [ -e "$LOCK_FILE" ]; then
    echo "Помилка: скрипт вже виконується"
    exit 1
fi

touch "$LOCK_FILE"
trap "rm -f $LOCK_FILE" EXIT
```

- Створюється lock-файл `/tmp/backup.lock`
- Якщо він вже існує — скрипт не запускається
- Команда `trap` гарантує видалення lock-файлу навіть при помилці

### 4. Створення архіву з міткою часу

```bash
TIMESTAMP=$(date +"%Y-%m-%d_%H-%M")
ARCHIVE_NAME="logs_backup_${TIMESTAMP}.tar.gz"
tar -czf "$ARCHIVE_PATH" -C "$LOGS_DIR" .
```

- Формується мітка часу у форматі `YYYY-MM-DD_HH-MM`
- Створюється архів `.tar.gz` з усіма файлами з каталогу логів
- Опція `-C` змінює робочий каталог перед архівацією

### 5. Перевірка результату

```bash
if [ $? -eq 0 ]; then
    echo "✅ Успіх!"
else
    echo "❌ Помилка"
    exit 1
fi
```

Перевіряється код виходу попередньої команди (`$?`). Якщо `0` — успіх, інакше — помилка.
